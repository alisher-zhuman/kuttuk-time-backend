# KuttukTime — Платформа подарочных сертификатов

## Что это?

TMA в Telegram для быстрой покупки и отправки подарочных сертификатов от локальных бизнесов (кофейни, спа, рестораны, салоны красоты).

**Главная идея:** Пользователь за 30 секунд покупает сертификат и отправляет его себе или в подарок. Мерчант получает уведомление и отслеживает использование.

---

## Flow для пользователя (4 шага, 30 секунд)

```
1. Открыл KuttukTime в Telegram
2. Выбрал мерчанта (Кофе Ali, Спа Вишня, Ресторан Мараба...)
3. Выбрал номинал (500, 1000, 2000 сом...)
4. Оплатил → получил код → поделился в Telegram
```

Регистрация: не нужна. Номер телефона: не нужен.

---

## Flow для мерчанта

**Подключение:** одна специальная ссылка → открыл в Telegram → система запомнила его как владельца заведения.

**Кабинет мерчанта показывает:**
- Статистика за неделю (продано N сертификатов, сумма, к выплате)
- Активные коды (список с кнопкой "Отметить использованным")
- Новые заказы в реальном времени

**Процесс использования сертификата:**
1. Клиент приходит, показывает код (KT-ALI-5847)
2. Кассир открывает приложение KuttukTime
3. Кликает "Отметить использованным" → код перходит в использованные
4. Клиент получил услугу ✅

**Еженедельные платежи (каждый понедельник):**
- Система автоматически считает статистику
- Отправляет отчёт мерчанту в Telegram
- Платит деньги на номер/карту через Finik/Bakai/Freedom

## Бизнес-модель

**Комиссия: 10%**

```
Пользователь платит:     1000 сом
Платёжная система:         -20 сом (2%)
Мерчанту:                 -900 сом (90%)
Наш доход:                 +80 сом (~8% чистыми)
```

**Прогноз:**
| Период | Мерчантов | Заказов/неделю | Доход/месяц |
|--------|-----------|----------------|-------------|
| Месяц 1-2 | 2 | 20-50 | 2,000-3,000 сом |
| Месяц 3-4 | 5-10 | 100-200 | 15,000-30,000 сом |
| Месяц 6+ | 20-50 | 500+ | 75,000+ сом |

## Операционный процесс

**Неделя 1-2:** встреча с мерчантом, обговорить номиналы, дать ссылку → готово.

**Каждый день:** смотреть новые заказы, отвечать на вопросы. Всё автоматически.

**Каждый понедельник:** проверить статистику (5 минут) → заплатить мерчантам.

На старте (2 мерчанта) = 2-3 часа работы в неделю.

## Архитектура Backend

**Stack:** NestJS + TypeORM + PostgreSQL + JWT

**Роли:** user (покупатель), merchant (бизнес), admin (владелец системы — захардкожен по Telegram ID)

**Auth:** `POST /auth/login { telegramId }` → JWT. Регистрация = авто при первом входе.

**Порядок разработки модулей:**
1. Users module ← текущий следующий шаг
2. Merchants module
3. Orders module (коды, генерация, статус)
4. Admin module
5. Payment integration (Finik/Bakai/Freedom)

**Production:** `https://kuttuk-time.koyeb.app` (Koyeb, Frankfurt, Free tier)
**CI/CD:** push в `main` → автоматический редеплой
